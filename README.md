
### 🔐 关于微信授权登录与手机号获取的说明

本项目已完美适配微信小程序最新的用户授权规范，实现了**“手机号一键授权登录 + 真实头像获取 + 真实昵称填写”**的完整身份认证闭环。

#### ⚠️ 开发与本地测试环境注意
由于微信官方接口规则限制，`<button open-type="getPhoneNumber">`（获取用户手机号）功能**仅对已认证的企业主体小程序开放**，且需要在微信公众平台后台开通服务。
如果你当前使用的是**个人开发者 AppID** 或是未开通该权限的测试账号，点击“手机号一键登录”时，会被微信底层直接拦截，无法弹出原生的微信授权底部半屏窗口。

为了保证开发阶段能够顺利体验和调试完整的 UI 交互（如登录前后的页面状态切换、资产数据渲染、修改真实头像等），我们在 `user.js` 中加入了**「模拟登录降级方案」**：
1. 当代码捕获到无权限报错（`getPhoneNumber:fail`）时，会自动弹出一个自定义的确认框。
2. 开发者点击**“确定”**后，系统将生成虚拟的手机号与随机昵称，并强制进入登录成功状态。
3. 登录成功后，即可正常体验点击头像调用 `chooseAvatar`、点击昵称调用 `type="nickname"` 键盘的完整流程。

#### 🚀 生产环境上线指南
本项目的核心登录逻辑已完全符合官方标准。当项目交付并准备上线时：
1. 只需在微信开发者工具右上角【详情】中，将 AppID 更换为**已认证的企业 AppID**。
2. 确保在微信公众平台后台开通了“获取手机号”权限。
3. **无需修改现有核心逻辑代码**，原生的微信手机号授权弹窗即可自动生效，无缝切换至生产环境。
